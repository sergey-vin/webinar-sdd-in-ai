# CUThere — Roadmap

## Фаза 1: Скелет
- [ ] Инициализация Next.js + Supabase + PWA + i18n (ru/en)
- [ ] Supabase Auth + 2 тестовых пользователя + переключатель в профиле
- [ ] Базовая схема БД: пользователи, маршруты, транспорт
- [ ] Навигация: главная, поиск, профиль
- [ ] Playwright: базовый smoke-тест

## Фаза 2: Маршруты
- [ ] Mock-данные: паромы, поезда, автобусы, самолёты (европейские маршруты)
- [ ] Поиск маршрутов: откуда — куда, дата
- [ ] Карточка маршрута: вид транспорта, время, цена
- [ ] Мультимодальные пересадки: отображение цепочки "паром + поезд + автобус"
- [ ] Часовые пояса: относительное время ("через 2ч 15мин"), локальное время порта

## Фаза 3: Пользовательские данные
- [ ] Карты лояльности: ручной ввод номера, привязка к профилю
- [ ] Сохранённые маршруты / избранное
- [ ] История поездок
- [ ] Офлайн-кеш маршрутов (PWA Service Worker)

## Фаза 4: Сравнение цен
- [ ] Mock-данные: цены на товары (выпивка, еда) на борту и в портах
- [ ] Экран сравнения цен: борт vs. порт
- [ ] Уведомления: "в следующем порту X дешевле на Y%"
- [ ] Апгрейд по милям: показ доступных апгрейдов на основе баланса карт лояльности

## Фаза 5: Полировка
- [ ] Сканирование карт лояльности (камера / NFC) вместо ручного ввода
- [ ] Push-уведомления: напоминания о рейсах, посадке
- [ ] Интеграция с реальными API перевозчиков (замена mock-данных)
- [ ] Performance-оптимизация, accessibility, финальные E2E тесты
