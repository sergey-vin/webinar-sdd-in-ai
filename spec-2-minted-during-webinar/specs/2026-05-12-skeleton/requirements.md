# Фаза 1: Скелет — Requirements

## Скоуп
Создание технического фундамента приложения CUThere: проект, авторизация, навигация, схема БД с заделом, визуальный стиль, тестовая инфраструктура. Код пока не пишем — только спеки.

## Принятые решения

### Репозиторий
- Пока создаём только спеки в `spec-driven/webinar/cruiseapp_new/specs/`
- Код будет в отдельном репозитории (решение отложено)

### Схема БД — с заделом
- Создаём таблицы не только для фазы 1, но и базовые для фаз 2-3: cities, routes, route_segments, transport_types
- Таблицы будут пустыми, заполнятся seed-данными в фазе 2
- Это позволит писать типы и интерфейсы уже в фазе 1

### Тестовые пользователи — разные роли
- `user_a` (user@cuthere.test) — обычный пользователь
- `user_b` (premium@cuthere.test) — премиум, с картами лояльности и балансом миль
- Роль хранится в `profiles.role` (enum: `user`, `premium`)
- Переключатель юзеров — только в dev-режиме

### Профиль — расширенный
- Имя, email, аватар
- Язык интерфейса (ru/en)
- Предпочтения транспорта (чекбоксы: паром, поезд, самолёт, автобус)
- Настройки уведомлений (on/off, заглушка — реальные push в фазе 5)
- Переключатель тестовых юзеров (dev)
- Кнопка выхода

### Навигация
- Bottom tabs на мобильном: Поиск, Мои маршруты, Цены, Профиль
- Top navbar на десктопе с теми же разделами
- Страницы Поиск / Мои маршруты / Цены — заглушки ("скоро будет")

## Отвергнутые решения
- **Минимальная схема БД (только profiles)** — отвергнуто, т.к. придётся переделывать типы и интерфейсы при добавлении таблиц в фазе 2
- **Одинаковые тестовые юзеры** — отвергнуто, т.к. разные роли нужны для тестирования фич лояльности (фазы 3-4)
- **Минимальный профиль** — отвергнуто, расширенный даст больше "мяса" для smoke-тестов и проверки i18n

## Контекст и заметки
- PWA: manifest.json + service worker заготовка, офлайн-кеш — в фазе 3
- i18n: структура `messages/ru.json` и `messages/en.json`, минимальные ключи для навигации и профиля
- Auth: только email+password, OAuth — потенциально позже
- Все страницы-заглушки должны быть стилизованы (не пустые div-ы), чтобы проверить тему/стиль
