# CUThere — Tech Stack

## Фронтенд
- **Next.js 15** (App Router) — SSR/SSG, роутинг, API routes
- **React 19** — UI-библиотека
- **TypeScript** — строгая типизация
- **shadcn/ui** (Radix + Tailwind) — UI-компоненты
- **Tailwind CSS** — стили, mobile-first
- **PWA** (next-pwa) — офлайн-доступ, установка на телефон
- **next-intl** — i18n (русский + английский)
- **Карта** (Leaflet / Mapbox) — интерактивная карта Европы для поиска маршрутов

## Бэкенд
- **Supabase** — PostgreSQL, Auth, Edge Functions, Realtime
- **Supabase Auth** — авторизация (email + OAuth)
- **Row Level Security** — доступ к данным на уровне БД

## Данные о маршрутах
- **Mock-данные** на первых фазах — SQL seed в Supabase БД
- Абстракция `TransportProvider` для будущей интеграции с реальными API

## Тестирование
- **Playwright** — E2E тесты
- **2 тестовых пользователя** с переключением в профиле (для удобства тестирования)

## Деплой
- **Vercel** — хостинг фронтенда
- **Supabase Cloud** — хостинг бэкенда

## Инструменты разработки
- **pnpm** — пакетный менеджер
- **ESLint + Prettier** — линтинг и форматирование
- **GitHub Actions** — CI/CD

## Дизайн-решения

### Визуальный стиль
- **Тон:** тёплый, дружелюбный (как Airbnb / Trainline)
- **Акцентный цвет:** коралловый/оранжевый (#FF6B6B), мягкие скруглённые углы, иллюстрации
- **Типографика:** скруглённый sans-serif (Inter / Nunito)

### Навигация
- **Bottom tabs** (мобильный паттерн): Поиск, Мои маршруты, Цены, Профиль
- На десктопе — те же разделы, верхняя навигация

### Поиск маршрутов
- Карта Европы + форма "откуда-куда-дата" рядом
- Результаты — **карточки** (не таблица)
- Карточка: иконки транспорта, время отпр./приб., итоговая цена, кол-во пересадок
- Сортировка: цена / время / пересадки. Фильтр: вид транспорта

### Детали маршрута
- **Компактный timeline:** одна строка на сегмент (иконка + время + город), раскрытие по клику
- При раскрытии: оператор, класс, платформа, длительность пересадки

### Прочее
- **UI-компоненты:** shadcn/ui (копируемые, полный контроль)
- **Mock-данные:** SQL seed в Supabase, не JSON-файлы

## Заметки
- Карты лояльности: на первой фазе — ручной ввод номера; сканирование/NFC — позже
- Часовые пояса: использовать `Intl.DateTimeFormat` + относительное время ("через 2 часа")
- Сравнение цен: mock-данные по ценам в портах и на борту
