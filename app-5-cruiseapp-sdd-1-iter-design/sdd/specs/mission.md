# Mission

## Продукт
CruiseApp — мобильное веб-приложение (responsive PWA) для пассажиров круизного лайнера.

## Проблема
Пассажир круиза сталкивается с хаосом: непонятно где какая палуба, расписание развлечений меняется, часы сбиваются при пересечении часовых поясов, а логистика до/после терминала — отдельная головная боль (парковка, такси, трансфер). Сейчас всё это — бумажные буклеты, разрозненные приложения и вопросы на ресепшн.

## Целевая аудитория
- Пассажиры круизных лайнеров (40-70 лет, средний и выше достаток)
- Не технические пользователи — интерфейс должен быть интуитивным
- Используют в основном смартфон (iPhone/Android), реже планшет

## Ключевые сценарии
1. **Бронирование каюты** — выбор каюты по палубе, типу, цене; просмотр на карте корабля
2. **Расписание развлечений** — что происходит сейчас / сегодня / завтра, фильтр по категориям (шоу, бассейн, ресторан, экскурсия), добавление в личное расписание
3. **Навигация по кораблю** — интерактивная карта палуб, поиск по названию (ресторан "Атлантика", бар "Закат"), маршрут от текущего местоположения
4. **Часовые пояса** — автоматическая подкрутка часов при смене пояса, отображение "корабельного времени" vs "домашнего времени", уведомление о переводе часов
5. **Логистика до/после терминала** — маршрут до порта (парковка, такси, общественный транспорт), информация о терминале, маршрут после высадки

## Тон и стиль
- Спокойный, дружелюбный, ненавязчивый
- Ощущение отпуска, не рабочего инструмента
- Минимум текста, максимум визуала

## Не-цели (чего НЕ делаем)
- Не делаем платёжную систему (бронирование — это выбор + запрос, оплата через внешний шлюз)
- Не делаем чат с экипажем
- Не делаем социальную сеть для пассажиров
- Не делаем систему управления для экипажа (только пассажирский интерфейс)
