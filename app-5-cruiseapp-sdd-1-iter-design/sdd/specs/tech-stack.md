# Tech Stack

## Frontend
- **Framework:** Next.js 15 (App Router)
- **Язык:** TypeScript (strict mode)
- **Стили:** Tailwind CSS
- **UI-компоненты:** shadcn/ui
- **Иконки:** Lucide React
- **Карта палуб:** SVG-based (не canvas, не image map)

## Дизайн
- **Подход:** mobile-first, responsive (320px — 1440px)
- **Цветовая палитра:** морская тема — тёмно-синий (#1a365d), белый (#ffffff), золотой акцент (#d4a574), светло-голубой фон (#f0f7ff)
- **Типографика:** Inter для UI, Playfair Display для заголовков
- **Скругления:** мягкие (border-radius: 12px для карточек, 8px для кнопок)
- **Тени:** лёгкие, не агрессивные
- **Анимации:** минимальные, плавные transitions (200-300ms)

## Данные
- **Состояние:** Zustand (простой, не требует boilerplate)
- **API:** REST (mock-данные на первом этапе, JSON-файлы в /public/data/)
- **Часовые пояса:** luxon (не moment.js — он deprecated)

## Архитектурные решения
- PWA: service worker для офлайн-доступа к расписанию и карте
- Все времена хранятся в UTC, отображаются в "корабельном времени" (настраивается)
- "Домашнее время" пользователя определяется при первом входе и сохраняется в localStorage
- Карта палуб — отдельный SVG на каждую палубу, зоны кликабельны
- Бронирование — stateless: выбор каюты → сводка → "Отправить запрос" (без платёжного шлюза)

## Ограничения
- Никаких серверных баз данных на первом этапе — всё в mock-данных
- Не использовать: jQuery, Bootstrap, Material UI, moment.js
- Не генерировать inline-стили — только Tailwind-классы
- Компоненты: один компонент — один файл, выносить типы в отдельные .ts файлы
