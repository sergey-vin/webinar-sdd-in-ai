# Mission

## Продукт
CUThere — мультимодальный travel companion (responsive PWA). Единое приложение для бронирования, навигации и сопровождения в поездках на паромах, самолётах, поездах и городском транспорте.

## Проблема
Путешественник использует десяток разрозненных приложений: Viking Line для парома, Finnair для рейса, HSL для трамвая, Google Maps для навигации. Каждое — свой аккаунт, свой интерфейс, свои уведомления. При этом:
- На корабле непонятно где какая палуба, расписание развлечений меняется
- Часы сбиваются при пересечении часовых поясов
- Логистика до/после терминала — отдельная головная боль
- Нет единого поиска маршрутов через разных перевозчиков (Google Maps знает не все паромы)

## Целевая аудитория
- Путешественники 25-70 лет, пользующиеся паромами/круизами между городами (Хельсинки, Таллин, Стокгольм, Санкт-Петербург)
- Также: авиапассажиры, пользователи городского транспорта в незнакомом городе
- Смартфон (iPhone/Android) — основное устройство, также desktop для планирования

## Ключевые сценарии
1. **Поиск и сравнение маршрутов** — кросс-перевозчик агрегатор (паромы, рейсы, поезда), фильтр по цене/времени/CO2, one-way и round-trip
2. **Планирование маршрута door-to-door** — от дома до конечной точки через все виды транспорта (такси -> метро -> паром -> трамвай)
3. **On-trip companion** — таймлайн поездки, boarding pass с QR, текущий статус (в море / в порту)
4. **Навигация по кораблю** — интерактивные SVG deck maps, поиск по venue, wayfinding
5. **Расписание развлечений** — TZ-aware (ship time / home time / local time), фильтр по категориям
6. **Часовые пояса** — автоматическое отображение в нужном TZ, уведомление о переводе часов
7. **Клубная карта** — tier tracking, очки, привилегии
8. **Профиль** — переключение демо-пользователей, настройки языка и таймзоны

## Тон и стиль
- Теплый, современный, спокойный
- Ощущение путешествия, не рабочего инструмента
- Минимум текста, максимум визуала
- Палитра: off-white #F4F1EB, harbor blue #0B3D5C, coral accent #E8623A

## UX-принципы для времени и навигации
- **Единый TZ-режим** — header, schedule, timeline, boarding pass — все показывают время в одном выбранном режиме (ship / home / local). Переключение через TZ toggle на странице расписания влияет глобально.
- **Всегда показывать TZ label** — рядом с каждым временем должна быть метка (EET, MSK, UTC+3). Пользователь никогда не должен гадать "в каком это часовом поясе?"
- **TZ change notification** — между этапами маршрута, если меняется часовой пояс, показывать pill-индикатор "EET → EEST"
- **Время до прибытия** — на boarding pass активного этапа показывать countdown "1h 23m remaining"
- **Локация событий** — у каждого события в расписании должны быть venue + deck, чтобы пользователь мог найти место на корабле

## Карта палубы и события на борту
- **Реалистичная SVG-карта** — каждая палуба: коридоры, зоны venue (прямоугольники с подписями), кабинные зоны (штриховка). Не generic outline, а plan этажа.
- **"Ваша каюта"** — маркер каюты пользователя на карте (парсится из seat_info: "Cabin 6021" → палуба 6). Пульсирующий teal маркер с 🛏. Только для паромных этапов.
- **Tap venue → события** — при нажатии на venue показываем карточку с предстоящими событиями в этом месте (время, название, save). Пользователь должен видеть не только "что тут", но и "что тут будет".
- **Сохранённые события на timeline** — закладки из расписания появляются между этапами маршрута на главном таймлайне как компактные карточки (иконка + название + время + venue + deck).

## Не-цели (чего НЕ делаем)
- Не делаем реальную платежную систему (бронирование — это выбор + просмотр, без оплаты)
- Не делаем чат с экипажем / другими пассажирами
- Не делаем систему управления для экипажа
- Не делаем реальную интеграцию с API перевозчиков (demo data in Supabase)
- Не делаем push-уведомления (PWA scope, без native app)
